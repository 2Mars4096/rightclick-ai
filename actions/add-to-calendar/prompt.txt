Rules:
- If the text has no schedulable event, return {"events":[],"reason":"short explanation"}.
- If the selected text contains multiple non-empty lines, treat each non-empty line as a separate candidate event.
- Preserve the original line order in the output events array.
- Do not merge separate lines into one event unless a single line clearly describes multiple events.
- Blank lines are separators only. Ignore them.
- If a date is present but no time is present, use an all-day event.
- For all-day events, use YYYY-MM-DD for start and end, and make end the day after the last covered day.
- If the text does not specify a timezone, use the current local timezone.
- If a line gives month/day and time but no year, infer the next upcoming matching date relative to the current local datetime.
- Keep the original event title wording when possible. Do not translate or rewrite titles unless needed.
- Short trailing venue abbreviations or codes like KG, GL, IS, PE should be preserved in location or notes instead of being dropped.
- Put leftover useful details into notes.
- Never wrap JSON in markdown fences.
